# DocFinder — AI 문서 검색 SaaS (포트폴리오 프로젝트)

여러 문서를 한꺼번에 올리면, 지정한 단어가 **어느 문서 몇 번째 줄**에 있는지 찾아서 **엑셀로 뽑아주는** 도구를 중심으로, 랜딩페이지·로그인·요금제·대시보드까지 갖춘 **AI SaaS 형태의 포트폴리오**입니다.
실사용자에게 판매하는 제품이 아니라, "AI로 SaaS를 만들고 실제로 배포까지 할 수 있다"는 걸 보여주기 위한 데모입니다.

> ⚠️ **포트폴리오 데모임을 밝힙니다.** 로그인은 실제 서버 인증/DB 없이 브라우저(localStorage)에 저장되는 방식이고, 결제는 Stripe **테스트 모드**로 연결되어 실제 청구가 발생하지 않습니다. 문서 검색과 AI 유사어 제안 기능 자체는 실제로 동작합니다.

## 페이지 구성

| 파일 | 역할 |
|---|---|
| `index.html` | 랜딩페이지 (제품 소개, CTA) |
| `signup.html` / `login.html` | 회원가입 / 로그인 — 이메일(mock) + Google 로그인(GSI) |
| `pricing.html` | Free / Pro 요금제 비교, Stripe Checkout(테스트 모드) 연결 |
| `dashboard.html` | 로그인 후 진입하는 SaaS 셸 — 사이드바(문서 검색 도구 / 사용량 / 설정) |
| `tool.html` | 실제 문서 검색 도구 본체. 단독으로 열어도 동작하고, 대시보드에 iframe으로도 임베드됨 |
| `shared/theme.css` | 전체 페이지가 공유하는 디자인 토큰·컴포넌트 |

| `api/synonyms.js` | Groq LLM으로 유사어 제안 (서버리스 함수) |
| `api/create-checkout-session.js` | Stripe Checkout 세션 생성 (서버리스 함수) |

## 문서 검색 도구의 주요 기능

- **여러 문서 동시 업로드** — PDF · Word(.docx) · Excel(.xlsx) · CSV · TXT (내 컴퓨터 또는 **구글 드라이브에서 직접**)
- **구글 드라이브 연결 + 자동 수집** — Google Picker로 문서를 직접 고르거나, **"최근 바뀐 문서"(어제부터·최근 3/7/30일)를 알아서 찾아** 골라 불러옴. `drive.readonly` 스코프(자동 수집엔 드라이브 읽기 권한 필요), Google 문서·시트는 PDF·xlsx로 자동 변환. 내려받은 문서 본문은 브라우저 안에서만 처리(서버로 안 나감)
- **스캔 이미지 PDF도 인식(OCR)** — 글자가 사진으로 박힌 스캔본을 자동 감지해서 `Tesseract.js`(한글+영어)로 읽어 검색 대상에 포함. 브라우저 안에서 처리(문서 본문은 서버로 안 나감)하고, 속도 때문에 기본은 꺼져 있다가 스캔본이 감지되면 켜도록 안내. 인식 엔진은 최초 1회만 내려받음
- **전체 문서 단어 검색** — 어느 문서 몇 번째 줄(PDF는 페이지, 엑셀은 시트!행)인지까지 표시
- **AI 유사어 제안** — "계약금" 입력 시 "선금·착수금·선수금"도 함께 찾을지 제안 (Groq LLM). 업로드된 문서가 계약서/견적서/발주서/세금계산서 중 어떤 유형인지 자동 추정해서 그에 맞는 유사어를 요청함
- **자동 폴백** — AI가 응답 못 하면(키 없음/한도초과/오프라인) 내장 사전으로 조용히 대체 → 앱이 죽지 않음
- **엑셀 다운로드** — 검색 결과를 정리된 표로 내보내기
- **정리 리포트** — 올린 문서들에서 "정리할 항목(사고·소식·계약·납품 등)"별로 **몇 건씩 어느 문서 어디에** 있는지 모아 요약+상세 2장짜리 엑셀로 출력
- **아침 리포트** — 위 조각들을 하나로: **어제 드라이브에서 바뀐 문서를 자동 수집 → 정리 → 엑셀**을 "지금 미리보기" 한 번으로 실행하고, 출근 시각·항목·이메일로 **예약 저장**. "아침에 오면 어제 퇴근 후 소식이 정리돼 있다"는 최종 비전의 실물 — **무인 자동 발송(사람 없이 정해진 시각에)만 서버 배포가 남음**
- **문서 유출 없음** — 파일 본문은 브라우저 안에서만 처리, 서버로 전송되지 않음 (API엔 검색어 단어만 전달)

## SaaS 형태로 감싼 부분

- **회원가입/로그인** — 이메일 mock 로그인 + Google Identity Services(진짜 구글 팝업)
- **요금제/페이월** — Free(월 검색 횟수 제한) / Basic / Middle / High 4단계 비교, 유료 플랜 업그레이드는 Stripe Checkout **테스트 모드**로 연결. 해당 플랜의 Price ID가 없으면 로컬에서 바로 데모 업그레이드로 폴백 (`shared/config.js`의 `PLANS`로 조절)
- **대시보드** — 사이드바 내비게이션, 이번 달 검색 사용량 게이지, 설정(플랜/구독 관리)
- **크레딧(토큰) 모델 · 용량 비례** — 기능 잠금 없이 **모든 기능은 전 플랜 공통**. **업로드는 무료·무제한**이고, 문서를 처음 처리할 때(첫 검색/정리/리포트) **용량만큼 딱 한 번** 차감 — 같은 문서 재검색·재정리는 추가 차감 없음. 기준은 `shared/config.js`의 `CREDIT_MODEL`(250KB당 1 크레딧, OCR 페이지당 소액 가산). 플랜은 월 크레딧 양으로만 구분(Free 100 · Basic 2,000 · Middle 6,000 · High 20,000, `PLANS[plan].credits`). 크레딧 소진 시 업그레이드 안내. (데모라 localStorage 기준 — 실제 과금은 서버 검증 필요)

## 기술 구성 (오픈소스 결합)

| 역할 | 사용 |
|------|------|
| PDF 텍스트 추출 | PDF.js |
| 스캔 이미지 PDF OCR | Tesseract.js (kor+eng, 브라우저 내 처리) |
| Word 텍스트 추출 | mammoth.js |
| Excel 읽기 / 엑셀 쓰기 | SheetJS (xlsx) |
| 유사어 제안 | Groq API (무료 등급) |
| 구글 로그인 | Google Identity Services |
| 구글 드라이브 문서 불러오기 | Google Picker API + Drive API (`drive.file`) |
| 결제 | Stripe Checkout (테스트 모드) |
| 배포 | Vercel (서버리스 함수 + 정적 호스팅) |

```
브라우저(문서 업로드)
   │  검색어 단어만 전송 (문서 본문 X)
   ▼
서버리스 함수(api/synonyms.js)  ← Groq API 키를 여기 숨김
   ▼
Groq API(문서유형 맞춤 프롬프트) → 유사어 반환   (실패 시 프론트가 내장 사전으로 폴백)
```

## 로컬에서 그냥 써보기 (배포 없이)

`index.html`을 브라우저로 열면 랜딩페이지부터 전체 흐름(가입 → 대시보드 → 문서 검색 → 요금제)을 볼 수 있습니다.
AI 유사어와 Stripe 결제는 서버리스 함수가 없으니 자동으로 **내장 사전 / 데모 업그레이드**로 대체됩니다.
`tool.html`만 단독으로 열어도 검색·엑셀 기능은 로그인 없이 그대로 동작합니다.

## 배포하기 (Vercel · 무료)

1. 이 폴더를 GitHub 저장소로 올린다
2. [vercel.com](https://vercel.com)에서 해당 저장소를 Import 한다
3. 필요한 환경변수/설정값은 **[SETUP-TODO.md](SETUP-TODO.md)** 에 체크리스트로 정리해뒀습니다 (Groq API 키, Google Client ID, Stripe 테스트 키 발급 방법 포함)
4. Deploy → 실제 URL이 생성된다

> ⚠️ **API 키(Groq, Stripe Secret Key)를 절대 HTML/프론트 코드에 넣지 마세요.** 브라우저 소스에 노출되어 도용됩니다. 반드시 Vercel 환경변수(서버)에만 두세요. `api/` 폴더 안의 서버리스 함수만 이 키들을 읽습니다. (단, Google Client ID는 원래 공개되는 값이라 `shared/config.js`에 직접 넣습니다.)

> ℹ️ 무료 LLM 등급은 rate limit과 모델 목록이 예고 없이 바뀔 수 있습니다. 모델이 사라지면 `GROQ_MODEL` 환경변수만 교체하면 되고, 아예 응답이 없어도 앱은 내장 사전으로 계속 동작합니다.

## 현재 한계 (다음 버전 로드맵)

- ~~**스캔 이미지 PDF**(글자가 사진인 문서)는 인식 안 됨~~ → ✅ `Tesseract.js` OCR(한글+영어)로 처리. 남은 과제: 손글씨·저품질 스캔은 인식률이 떨어지고, 첫 페이지는 엔진 다운로드로 느림 → 워커 재사용/전처리(이진화·기울기 보정)로 개선 여지
- ~~**구글 드라이브 직접 연결** 미지원~~ → ✅ Google Picker로 직접 고르기 + **최근 바뀐 문서 자동 수집**(`drive.readonly`, 브라우저 내 처리)까지 구현. 남은 과제: **퇴근~출근 시간에 맞춰 이 수집·정리를 무인으로** 도는 자동화 — 이건 서버 예약작업(Cron)·저장된 드라이브 토큰(오프라인 접근)·요약(LLM)·메일이 필요해 정적 데모 밖. 최종 단계로 남김
- 내장 사전은 한국어 비즈니스 용어 위주 → AI 폴백이 이를 보완
- 로그인/결제는 실제 서버 검증이 없는 데모 구현 — 프로덕션이라면 서버 세션 인증 + Stripe 웹훅 검증이 필요
