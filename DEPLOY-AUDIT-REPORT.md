### 2. `login.html` · `signup.html` — 375px에서 가로 스크롤 51px
- **실측:** 뷰포트 375 / 문서너비 **426** / 넘침 **51px**. 두 페이지 동일. (iframe 아니라 진짜 375 뷰포트에서 직접 로드.)
- **뿌리:**
  ```
  .authsplit      display:grid   grid-template-columns 계산값 = 425.6px   (컨테이너는 375)
    └ .authform-col   min-width: auto   폭 426   (padding 40px 24px)
        └ .authcard   width:100% max-width:400px   폭 377.6
  ```
  `@media (max-width:820px){ .authsplit{ grid-template-columns:1fr } }`는 **적용됐다.**
  그런데 **`1fr` = `minmax(auto, 1fr)`**이고, 그리드 아이템의 기본 `min-width:auto` 때문에
  트랙이 아이템 min-content(425.6px) 아래로 못 내려간다. 교과서적 그리드 오버플로.
- **표준 해법(적용 안 함):** `.authsplit{ grid-template-columns: minmax(0,1fr) }` 또는 `.authform-col{ min-width:0 }`.
  ⚠️ 고친 뒤 **375px 재측정 필수.** `min-width:0`을 주면 이번엔 `.authcard` 안쪽(min-content 377.6px짜리
  무엇)이 넘칠 수 있는데, **그 안쪽 범인은 못 짚었다.**
- **오늘 것과 무관:** `6174cb6`에도 `.authform-col` 존재. 오늘 `signup.html`에 넣은 건 푸터 2줄이 전부.

## 🟡 이상한 것 (확신 낮음 / 관문 밖)
1. **폰트 3,063KB** (라이브 캐시 우회 실측: Regular 748 · SemiBold 767 · Bold 773 · ExtraBold 775).
   서브셋 안 된 Pretendard 원본 4개. **회의 3인 만장일치로 "지금 하지 말 것"** — 관문 밖 골드플레이팅.
2. **폰트 `cache-control: public, max-age=0, must-revalidate`.** 절대 안 바뀌는 정적 자산인데 매 요청 재검증.
   `immutable, max-age=31536000`이 표준. **확신 낮은 이유:** 대역폭은 304로 아껴지고 서비스워커가 덮어서
   실제 사용자 체감 영향을 못 쟀다. 관문도 아님.
3. **`tool.html`에 `h1` 4개.** 모드별로 하나씩인 듯하고 한 번에 하나만 보일 가능성이 높다.
   **확신 낮은 이유: 각 h1의 가시성을 안 쟀다.** 앱 화면이라 SEO 영향 거의 없음.
